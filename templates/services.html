Data Collection

PHISHING URLs
Using the services of Phishtank has allowed us to obtain a dataset of phishing urls. Phishtank provides dataset of most recently detected phishing url in form of csv, json, xml etc. Phishtank is basically a community site where anyone can submit, verify, track and share phishing data.
It consists of following columns - phish_id, url, phish_detail_url, submission_time, verified, verification_time, online, target.

LEGITIMATE URLs
The legitimate URLs are obtained from the open datasets of the University of New Brunswick, which contains legitimate URLs from year 2016 and onwards. This dataset has a collection of benign, spam, phishing, malware & defacement URLs. Out of all these types, the benign URL dataset is considered for this project. From this dataset, 10,000 random legitimate URLs are collected to train the ML models.

Model Creation

HYBRID MODEL
This model is a combination of two or more algorithms to obtain a model that covers all the best features of both algorithms used. This model is developed in 2 phases, in the first phase we pick our machine learning algorithms and apply it to our dataset and on evaluation we select 4 best performing algorithms. In the second phase we find the best combination of algorithms we got in phase-1 to overcome individual limitations. The first step of model creation requires us to split the data.
Train-Test Split: Initially, we split the data into features and target set. Data in 'X' variable consists of the features which will be used by our model to perform predictions. Data in 'y' variable is the target label. The ratio of size of training set to testing set is (75:25) %.
From our dataset it was clear that this machine learning task was supervised in nature. Under supervised learning there are two types namely, classification and regression. Phishing URL detection falls under classification problem, i.e., the input of the URL and target label is 0 (legitimate) and 1 (phishing).
Therefore, the machine learning models (classifiers) used to train this dataset are:
1. Decision Tree
2. Random Forest
3. Support Vector Machine (SVM)
4. XGBoost
5. Gradient Boosting Machine
6. Adaboost
7. Multi-Layer Perceptron (MLP) - Neural Network

To evaluate the performance of our machine learning models, we have used the following evaluation metrics:
1. Classification report - The classification report visualizer displays the precision, recall, F1, and support scores for the model.
2. Precision - Precision is the ability of a classifier not to label an instance positive that is actually negative. For each class, it is defined as the ratio of true positives to the sum of a true positive and false positive.
3. Recall - Recall is the ability of a classifier to find all positive instances. For each class it is defined as the ratio of true positives to the sum of true positives and false negatives.
4. F1 score - The F1 score is a weighted harmonic mean of precision and recall such that the best score is 1.0 and the worst is 0.0.
5. Support - Support is the number of actual occurrences of the class in the specified dataset.

Confusion Matrix is a performance measurement for machine learning classification problem where output can be two or more classes. It is a table with 4 different combinations of predicted and actual values. It is extremely useful for measuring Recall, Precision, Specificity, Accuracy, and most importantly AUC-ROC curves.
1. True Positive: You predicted positive and it’s true.
2. True Negative: You predicted negative and it’s true.
3. False Positive (Type 1 Error): You predicted positive and it’s false.
4. False Negative (Type 2 Error): You predicted negative and it’s false.

ROC AUC Curve:
The Receiver Operator Characteristic (ROC) curve is an evaluation metric for binary classification problems. It is a probability curve that plots the TPR against FPR at various threshold values and essentially separates the ‘signal’ from the ‘noise’.
The Area Under the Curve (AUC) is the measure of the ability of a classifier to distinguish between classes and is used as a summary of the ROC curve. It tells how much the model is capable of distinguishing between classes. Higher the AUC, the better the model is at predicting 0 classes as 0 and 1 classes as 1. An excellent model has AUC near to the 1 which means it has a good measure of separability. The AUC score can be computed using the roc_auc_score() method of sklearn.

Hyperparamter Tuning


Comparison of Methods
The top 4 models based on their evaluation metrics are Decision Tree, Random Forest, XGBoost and MLP. However, due to its high computational complexity we will not be considering MLP for preparing our ensemble. Instead, we will make use of the Gradient Boosting Machine which is an ensemble and the next best performing model.

Hyperparameter Tuning
A hyperparameter is a parameter whose value can be used to control the learning process. In machine learning, hyperparameter tuning is the process of choosing a set of optimal hyperparameters for a learning algorithm.
Methods of Hyperparameter Tuning (scikit-learn):
In both the methods we create a grid of possible values of hyperparameters.
Grid Search - Grid Search iterates through the grid and tries the combinations of hyperparameters in a specific order. Then, it fits the model on every combination and records the performance. The result is the best model with the best hyperparameters.
Random Search - Random Search iterates through the grid, tries random combinations of hyperparameters and provides the best model with best hyperparameters.
The selected models DTC, RFC, XGB and GBM, underwent the process of hyperparameter tuning to improve their accuracy.

Hybrid Model


Ensemble Model
Ensemble methods are meta-algorithms that combine several machine learning techniques into one predictive model in order to decrease variance (bagging), bias (boosting), or improve predictions (stacking).
For our project we have considered two ensemble methods:
1. Stacking
2. Voting Classifier
Bagging and Boosting techniques are aimed to decrease variance and bias, however our aim was to combine machine learning models in order and improve accuracy and boost performance when tested against unseen tested, which is why we decided on methods that help improve accuracy i.e., Stacking and Voting Classifier.
Stacking Classifier
Stacking is an ensemble learning technique that combines multiple classification or regression models via a meta-classifier or a meta-regressor. For stacking we used StackingClassifier from sklearn. Stacked generalization consists in stacking the output of individual estimator and use a classifier to compute the final prediction. Stacking allows to use the strength of each individual estimator by using their output as input of a final estimator.
Voting Classifier
A Voting Classifier is a machine learning model that trains on an ensemble of numerous models and predicts an output (class) based on their highest probability of chosen class as the output. It simply aggregates the findings of each classifier passed into Voting Classifier and predicts the output class based on the highest majority of voting. The idea behind this was that, instead of creating separate dedicated models and finding the accuracy for each them, we create a single model which trains by these models and predicts output based on their combined majority of voting for each output class.
Voting Classifier supports two types of voting:
1. Hard Voting: In hard voting, the predicted output class is a class with the highest majority of votes i.e., the class which had the highest probability of being predicted by each of the classifiers.
2. Soft Voting: In soft voting, the output class is the prediction based on the average of probability given to that class.

Final Model
After performing a series of combinations between the models we have trained on our dataset, we came to the conclusion that the Voting Classifier using base estimators – Decision Tree, XGBoost and Gradient Boosting Machine is the best model to classify the URL of a phishing website.
